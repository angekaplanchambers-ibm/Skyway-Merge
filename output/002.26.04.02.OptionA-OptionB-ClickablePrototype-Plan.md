## What This Plan Covers

Create one clickable prototype that supports two IA variants from the Skyway merge diagram:

- **Option A** - Org-first (diagram section: "Option A - Long-Tail Org-First")
- **Option B** - Workspace-centric (diagram section: "Option B - Skyway Workspace-Centric")

The prototype focuses on landing screens only, with one buried path:

- `Workspace -> Search/Import`

## Scope

### In scope

- Two variants in one prototype: Option A and Option B
- Landing screens only for each variant
- Search/Import as the only buried path
- Shared dummy data between variants for fair comparison

### Out of scope

- Deep drill-down pages
- Backend integration
- Policy engine implementation

## Screen Inventory

### Option A screens (Org-first)

1. Option A org dashboard landing (`/app/{org}/option-a`)
2. Org explorer landing (`/app/{org}/explorer`)
3. Org infragraph landing (`/app/{org}/infragraph`)
4. Org settings landing with IAM visible (`/app/{org}/settings`)
5. Workspace landing (`/app/{org}/workspaces/{ws}`)
6. Workspace Search/Import landing (`/app/{org}/workspaces/{ws}/search`)

### Option B screens (Workspace-centric)

1. Org selection (`/app/organizations`)
2. Option B org dashboard (`/app/{org}/option-b`)
3. Workspace overview landing (`/app/{org}/workspaces/{ws}`)
4. Workspace explorer landing (`/app/{org}/workspaces/{ws}/explorer`)
5. Workspace infragraph landing (`/app/{org}/workspaces/{ws}/infragraph`)
6. Workspace Search/Import landing (`/app/{org}/workspaces/{ws}/search`)

## Interaction Rules

- Every landing entry point is clickable.
- Keep navigation shallow.
- Only deep path allowed: `workspace -> search/import`.
- Use same dummy data across Option A and Option B.
