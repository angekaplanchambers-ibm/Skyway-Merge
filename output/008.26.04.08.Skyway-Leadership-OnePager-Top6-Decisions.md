# Skyway Leadership One-Pager: Top 6 Meeting-Derived Decisions

**Date**: 2026-04-08  
**Audience**: Product, Design, Engineering Leadership  
**Scope**: Organization Portfolio and Organization Overview decisions derived from 01.meetings

## Executive Summary
This one-pager reflects the same decision set defined in the Rationale Matrix, anchored to 01.meetings evidence.

The operating model remains a connected 3-step journey:

1. **Find / Investigate / Be alerted**
2. **View / Understand context**
3. **Remediate / Take action**

## Top 6 Decisions and Impact

| Decision | Why this decision was made (from meeting evidence) | Source evidence (01.meetings) | Leadership impact statement |
|---|---|---|---|
| **1. Treat the experience as one connected 3-step journey: Find/Investigate -> View/Understand -> Remediate/Act.** | Workshop materials describe overlap across discovery, context, and remediation steps, requiring continuity across surfaces. | "The group aligned on a three-stage infrastructure journey that all three tools partially support: Find / Investigate, View / Understand, Remediate / Act." (TRANSCRIPT-Skyway Merge Design Workshop.md Summary sections 7 and 8) | Scope decisions stay cleaner: shared interaction patterns can be standardized without forcing full functional merger. |

## Readout for Leadership

- The decision frame is evidence-first and tied to 01.meetings, not implementation convenience.
- The strategy supports consistency where it reduces friction and separation where it protects task clarity.
- The 3-step model provides a common rubric for product, design, and engineering prioritization.

## Current Wireframe Implications

- **Decision 3 is now explicit in the UI**: Organization Overview now spells out the role of Explorer View, Resource Graph, and Search and Import so users can distinguish managed posture, relationship analysis, and import/remediation entry without guessing.
- **Decision 4 is now shown as behavior, not just language**: remediation queues now use concrete handoff links so the cross-surface workflow is visible and predictable.
- **Decision 3 is now represented at portfolio level**: Organizations Portfolio includes cross-organization relationship insights so InfraGraph's dependency/blast-radius value is visible in the overall experience.
- **Decision 5 now appears earlier in the flow**: unmanaged resource-type visibility is surfaced in portfolio coverage so Search and Import can be discovered from the same investigation path.
- **Decision 5 now includes an explicit conversion strategy**: a "Convert to Managed" CTA in "Top unmanaged types" deep-links to Search and Import with resource-type context, making unmanaged-to-managed remediation a direct next step.
- **Decision 6 changed what was removed, not just what was added**: the standalone source-freshness panel was removed because it added scan cost without materially improving wayfinding or action decisions.
