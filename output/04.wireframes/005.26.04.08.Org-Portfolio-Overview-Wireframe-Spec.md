## Intent

Define a concrete, implementation-ready wireframe spec for an all-organizations view where one user can see posture, risk, and remediation entry points across every organization they can access.

## Route and Scope

- Primary route: `/app/organizations`
- Primary user: Org admin, platform operator, governance lead
- Time windows used by widgets: `24h`, `7d`, `30d`
- Data scope: all organizations where user has read access

## Filters, Sorting, and Query Params

- Search: `q` (organization name, ID, tag)
- Severity filter: `severity=critical|high|medium`
- Health filter: `health=healthy|warning|critical`
- Deploy mode filter: `deployMode=tfe_on_prem|hcp_saas`
- Sort options:
  - `name_asc`
  - `health_desc`
  - `open_remediation_desc`
  - `freshness_asc`
  - `last_successful_run_desc`

## Action Mapping (Insight -> Action)

- Drift issues -> `openRunDetail` or `openWorkspaceHub`
- Validation failures -> `openWorkspaceHub` (health/config)
- Unmanaged resources -> `openSearchImport`
- Dependency hotspot -> `openResourceGraph`
- Governance issue -> `openExplorer` or org settings route

## Empty, Loading, and Error States

- Empty organizations: show onboarding CTA and permissions hint.
- Loading: skeleton for Z3-Z8; retain header and filters.
- Partial source degradation: show stale badge by source and degraded banner.
- Permission mismatch: mask unauthorized org rows and show count of hidden organizations.

## Responsive Behavior

- Tablet: collapse Z6 into stacked cards, preserve Z5 and Z8 full width.
- Mobile: KPI cards horizontal scroll; queue and table become card lists with same field names.

## QA Checklist

- Every row in Z8 has a deterministic `primaryActionRoute`.
- All queue items in Z5 map to one of the five `nextActionType` values.
- Source freshness is visible at page level and row level.
- Managed/unmanaged counts reconcile: sum of row counts equals global totals (within scope/filter).
- Sorting and filter chips apply consistently to both Z5 and Z8 when relevant.
