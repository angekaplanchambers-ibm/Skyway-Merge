# Chat UI Microinteraction Patterns for Agent Work

## Scope

Define interaction-level behavior for a chat interface where an agent performs long-running work, may request input/approval, can retry on transient failures, and supports cancellation.

## Interaction principles

1. Acknowledge submit within 100 ms.
2. Show one primary run status at a time.
3. Preserve user input and generated output across retry/reconnect.
4. Use bounded waiting with elapsed time and clear next action.
5. Provide one clear recovery path for each failure.
6. Gate external or destructive actions behind explicit approval.
7. Keep long-running work cancelable.
8. Ensure keyboard, screen reader, and reduced-motion support in every state.

## State inventory

### Composer states
- `idle`
- `drafting`
- `submit_pending`
- `send_failed_local`

### Agent run states
- `acknowledged`
- `planning`
- `running`
- `streaming_update`
- `waiting_external`
- `waiting_user_input`
- `approval_required`
- `retry_backoff`
- `cancel_pending`
- `cancelled`
- `completed`
- `partial_success`
- `failed_retryable`
- `failed_terminal`
- `timed_out`

### Session overlay states
- `connected`
- `reconnecting`
- `offline`

## Core transitions

| From | Event | To | UI response | Timing |
|---|---|---|---|---|
| `idle` / `drafting` | Send | `submit_pending` | Disable send, show pending user bubble | <=100 ms |
| `submit_pending` | Server ack | `acknowledged` | Show "Agent started" status | <=2 s |
| `submit_pending` | Ack timeout | `send_failed_local` | Inline error and Retry | at 2 s |
| `send_failed_local` | Retry | `submit_pending` | Re-send same payload | immediate |
| `acknowledged` | Plan ready | `planning` | Show plan summary row | <=500 ms |
| `planning` | Work starts | `running` | Activate step list | immediate |
| `running` | Stream/tool updates | `streaming_update` | Append incremental output | 100-300 ms batches |
| `running` / `streaming_update` | Need user input | `waiting_user_input` | Focus composer, show suggested replies | <=100 ms |
| `running` / `streaming_update` | Approval required | `approval_required` | Show approve/reject card | <=100 ms |
| `approval_required` | Approve | `running` | Resume and mark approval accepted | immediate |
| `approval_required` | Reject | `cancelled` | End run with rejection reason | immediate |
| `running` / `streaming_update` | Transient error | `retry_backoff` | Show countdown and retry counter | immediate |
| `retry_backoff` | Auto/manual retry | `running` | Resume current step | per backoff policy |
| `retry_backoff` | Retries exhausted | `failed_retryable` | Show Manual Retry and Cancel | after 3 tries |
| `running` / `streaming_update` | Stop clicked | `cancel_pending` | Disable Stop and show cancelling | immediate |
| `cancel_pending` | Cancel confirmed | `cancelled` | Show final cancelled state | <=2 s |
| `running` / `streaming_update` | Completed with no issues | `completed` | Show summary and outputs | immediate |
| `running` / `streaming_update` | Completed with warnings | `partial_success` | Show warnings and outputs | immediate |
| `connected` | Socket lost | `reconnecting` | Banner plus reconnect indicator | <=1 s |
| `reconnecting` | Link restored | previous run state | Replay missed events | <=30 s |
| `reconnecting` | Restore timeout | `offline` | Offline banner and reconnect action | at 30 s |

## Pattern checklist

1. **Submit acknowledgment** - immediate local pending state and server ack timeout handling.
2. **Progressive run feedback** - phase label, elapsed timer, active step.
3. **Step-level visibility** - each tool/action step has start, success, failure, retry state.
4. **User handoff** - explicit wait-for-input state with focus management.
5. **Approval gate** - high-impact action requires explicit approve/reject.
6. **Bounded retry** - max 3 retries, then fallback to manual decision.
7. **Cancel handshake** - request acknowledged immediately, confirmed when backend stops.
8. **Completion handoff** - output summary plus next actions.

## Accessibility behavior

- Keyboard: all primary actions reachable by Tab order; Enter sends; Shift+Enter inserts newline.
- Screen reader: polite announcements for phase change; assertive announcements for errors/timeouts.
- Focus: keep focus in composer during run unless a blocking approval card appears.
- Reduced motion: replace animated transitions with instant state swap and static indicators.
